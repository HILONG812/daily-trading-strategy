## 一、当前产品状态

### 1.1 产品概述
- **产品名称**: AI 模拟交易
- **定位**: 10 万美金虚拟账户，跟踪美股 + 港股 + 日股交易
- **目标用户**: 对股票投资感兴趣的学习者、想测试交易策略的人
- **当前形态**: 静态 HTML 页面 + JSON 数据存储

### 1.2 现有功能
| 功能模块 | 状态 | 说明 |
|----------|------|------|
| 账户概览 | ✅ | 显示初始资金、当前净值、今日盈亏、剩余现金 |
| 持仓展示 | ✅ | 按市场分组 (美股/港股/日股)，显示数量、成本、RSI 状态 |
| 交易记录 | ✅ | 历史买卖记录，含价格、数量、理由 |
| 数据汇总 | ✅ | 总投入、现金、持仓数量、市场分布 |
| 数据存储 | ✅ | trades.json 结构化存储 |

### 1.3 技术架构
```
trading.html (静态页面)
    ↓
trades.json (数据源)
    ↓
手动更新 (无自动化)
```

## 二、用户需求分析

### 2.1 核心需求 (基于产品设计推断)
1. **追踪投资表现** - 想知道账户赚了还是赔了
2. **理解交易逻辑** - 想知道为什么买这只股票
3. **跨市场对比** - 美股/港股/日股表现对比
4. **学习投资方法** - 通过 AI 的交易理由学习分析思路

### 2.2 潜在需求 (待验证)
1. **实时数据** - 当前页面盈亏"待结算"，用户可能想知道实时浮动盈亏
2. **图表可视化** - 资产变化趋势、持仓占比饼图
3. **交易统计** - 胜率、平均收益、最佳/最差交易
4. **提醒功能** - RSI 超买/超卖提醒、财报日期提醒
5. **导出分享** - 生成报告分享给朋友
6. **策略回测** - 如果当时没卖/多持有了会怎样

### 2.3 用户痛点 (推测)
| 痛点 | 严重程度 | 说明 |
|------|----------|------|
| 数据不实时 | 🔴 高 | 盈亏"待结算"，无法看到当前市值变化 |
| 无趋势图表 | 🟡 中 | 看不到资产曲线、持仓变化历史 |
| 手动更新 | 🟡 中 | 需要手动编辑 HTML 和 JSON，容易出错 |
| 无搜索筛选 | 🟢 低 | 交易多了之后难以查找特定股票记录 |
| 无移动端优化 | 🟢 低 | 已有基础响应式，但可进一步优化 |

## 三、数据展示优化方案

### 3.1 当前展示方式评估
**优点**:
- 简洁清晰，信息密度适中
- 按市场分组符合用户认知
- 颜色区分盈亏 (绿涨红跌)
- 包含交易理由，有教育价值

**缺点**:
- 缺少图表可视化
- 无时间维度 (资产曲线)
- 无对比功能 (vs 大盘指数)
- 盈亏数据不实时

### 3.2 建议新增图表

#### 3.2.1 资产曲线图 (必做)
```
类型：折线图
X 轴：日期
Y 轴：账户净值 ($)
辅助线：初始资金线、标普 500 指数 (可选)
价值：直观看到投资表现趋势
```

#### 3.2.2 持仓占比图 (必做)
```
类型：饼图/环形图
数据：各股票市值占比
交互：点击扇区显示详情
价值：了解风险集中度
```

#### 3.2.3 市场分布图 (选做)
```
类型：堆叠柱状图
X 轴：日期
Y 轴：各市场持仓金额
价值：观察市场配置变化
```

#### 3.2.4 交易日历 (选做)
```
类型：日历热力图
颜色深浅：当日交易金额/盈亏
价值：快速定位活跃交易日期
```

### 3.3 表格优化建议

#### 当前持仓表增强
| 字段 | 当前 | 建议新增 |
|------|------|----------|
| 股票名称 | ✅ | - |
| 数量/成本 | ✅ | - |
| 当前价 | ❌ | ✅ 实时价格 |
| 市值 | ❌ | ✅ 当前市值 |
| 浮动盈亏 | ❌ | ✅ 金额 + 百分比 |
| 持仓占比 | ❌ | ✅ 占账户比例 |
| RSI 状态 | ✅ | - |
| 买入理由 | ✅ | - |

#### 交易记录表增强
| 字段 | 当前 | 建议新增 |
|------|------|----------|
| 日期 | ✅ | - |
| 操作 | ✅ | - |
| 股票/价格/数量 | ✅ | - |
| 理由 | ✅ | - |
| 后续表现 | ❌ | ✅ 持有 X 天后涨跌 |
| 交易标签 | ❌ | ✅ #超卖 #龙头 #财报 等 |

## 六、本周优化建议 (Week 1)

### 🎯 核心目标：实现数据自动化

**问题**: 当前页面盈亏显示"待结算"，数据需要手动更新

**建议方案**:
1. 创建 `update_prices.py` 脚本
   - 使用 yfinance 获取持仓股票实时价格
   - 计算当前市值和浮动盈亏
   - 更新 trades.json

2. 修改 trading.html 模板
   - 将硬编码数据改为模板变量
   - 使用 Python + Jinja2 生成最终 HTML

3. 设置定时任务
   - 每天收盘后自动运行 (美股约凌晨 4 点)
   - 生成新页面并覆盖旧文件

**预期效果**:
- ✅ 实时显示浮动盈亏
- ✅ 减少手动操作错误
- ✅ 为后续图表功能打基础

**工作量估算**: 4-6 小时
